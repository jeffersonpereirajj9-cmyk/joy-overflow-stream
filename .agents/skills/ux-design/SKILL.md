Apply UX design principles to this project. Analyze the product and provide a structured UX review.

1. USER FLOWS — Map the 3 most critical user flows (e.g., onboarding, core task, error recovery). For each: list the steps, identify drop-off points where users are likely to get confused or abandon, and suggest simplifications.

2. INFORMATION ARCHITECTURE — Is the content organized in a way that matches user mental models? Check: navigation labels (are they user-language or company-language?), content grouping (related items grouped together?), search functionality (does it exist where users need it?).

3. FIVE-PLANE MODEL REVIEW — Quickly assess each plane: (1) Strategy: is the product goal clear to users on first visit? (2) Scope: are features prioritized by user need? (3) Structure: is the interaction model consistent? (4) Skeleton: is the layout scannable (F-pattern or Z-pattern)? (5) Surface: does visual design support the UX?

4. UX LAWS APPLICATION — Check compliance with key UX laws: Fitts's Law (are click targets large enough, min 44×44px?), Hick's Law (too many choices on any screen?), Miller's Law (more than 7±2 items in any list without grouping?), Jakob's Law (does the app follow common patterns users know?).

5. VALIDATION GAPS — What has not been user-tested? Identify the top 3 assumptions in the current design that carry the most risk if wrong, and suggest quick validation methods (5-second test, task-based usability test, click-map analysis).

For each issue: specific location in the app, the UX principle violated, and the recommended fix.
