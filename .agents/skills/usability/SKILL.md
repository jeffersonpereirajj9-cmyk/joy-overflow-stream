Run a usability evaluation on this project using Nielsen's 10 Heuristics and conversion optimization principles.

1. HEURISTIC EVALUATION — Check each of Nielsen's 10 heuristics and flag violations:
H1 Visibility of System Status: does the app always show what is happening (loading states, progress, success/error feedback)?
H2 Match with Real World: are labels and icons using language the user knows?
H3 User Control: can users undo actions, go back, and cancel processes?
H4 Consistency: are similar elements always styled and behaving the same way?
H5 Error Prevention: are there confirmations for destructive actions?
H6 Recognition over Recall: is all needed information visible (not hidden in menus)?
H7 Flexibility: are there shortcuts for power users?
H8 Aesthetic and Minimal Design: is there unnecessary information cluttering the interface?
H9 Error Messages: are errors described in plain language with a solution?
H10 Help: is contextual help available for complex features?

2. FORM USABILITY — For every form in the app: are labels above inputs (not just placeholders)? Is validation shown inline as the user types, not just on submit? Are required fields marked? Is the submit button disabled until the form is valid?

3. LOADING STATES — Does every async operation show a loading state? Are skeleton screens used instead of spinners where possible? Is there a timeout state if loading takes more than 10 seconds?

4. CONVERSION OPTIMIZATION — Identify the single most important conversion action in the app and audit the entire flow: How many steps? How many form fields? Is there social proof near the CTA? Is the value proposition clear on the conversion page?

Report each violation with: heuristic number, location, severity (1-4 scale), and specific fix.
